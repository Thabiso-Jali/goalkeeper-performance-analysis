## Project Overview

Goalkeeper is the hardest position to scout purely on data. Unlike outfield players, there are no clear universal metrics that predict whether a keeper will make it at a higher level. This project investigates that question systematically using Impect's KPI and player score data.

**Two prediction targets were modelled:**

| Target | Positive class | Question |
|--------|---------------|----------|
| `y_plays` | PLAYS (≥5 matches at higher level) | Will he become a regular starter? |
| `y_up` | PLAYS + BENCH | Is he ready to move up at all? |

`y_up` is used as the **main scouting model** — it captures transfer-readiness rather than mixing keeper quality with playing-time decisions made by the destination club.

## Methodology

The notebook follows five steps:

### Step 1 · Load Data
- Read labels from `gk_dataset_final.csv`
- Build a match-folder index (~10,765 folders) for O(1) lookup
- Load `player_scores.json` per match, filtered to `position == GOALKEEPER` and `matchShare ≥ 0.5`
- Cross-reference `player_kpis.json` to confirm position (since `player_scores` has no position field)

### Step 2 · Feature Engineering
- Aggregate per-keeper mean and std across all qualifying origin matches
- Two binary targets: `y_plays` and `y_up`
- Exploratory box plots by status category

### Step 3 · Feature Selection
Feature selection runs **twice** — once per target — using a shared helper:

1. **Curated GK feature list** — 21 GK-relevant score IDs (aerial, shot stopping, distribution, sweeping). Avoids outfield noise (e.g. OPEN_GOAL_SHOT_SCORE).
2. **Missing-value filter** — drop columns missing in >40% of keepers (GK_PREVENTED_GOALS_* only available from 2025–2026)
3. **Mutual information ranking** — rank remaining features against the target
4. **Correlation deduplication** — drop the weaker of any pair with |r| > 0.85

This produces `X_plays` / `feature_cols_plays` and `X_up` / `feature_cols_up`.

### Step 4 · Modelling
Three models trained on **each** feature set (6 models total):

| Model | Class balancing |
|-------|----------------|
| Logistic Regression | `class_weight='balanced'` |
| Random Forest | `class_weight='balanced'`, `min_samples_leaf=3` |
| XGBoost | `scale_pos_weight = neg/pos` |

Evaluation: stratified 5-fold cross-validation, metrics: AUC-ROC, Average Precision, F1, Precision, Recall.

### Step 5 · Validation
- Per-fold stability check (are scores consistent across folds?)
- Per-class classification report (out-of-fold)
- Train vs validation AUC gap (overfitting check)

## Key Findings

**Top predictive features** (by mutual information with `y_plays`):

| Rank | Feature | MI Score | Interpretation |
|------|---------|----------|----------------|
| 1 | GK_CAUGHT_AND_PUNCHED_HIGH_BALLS_PERCENT | 0.064 | Aerial dominance is the #1 signal |
| 2 | GK_PREVENTED_GOALS_HEADER_SHOT_RATIO | 0.036 | Shot stopping on headers |
| 3 | RATIO_PASSING_ACCURACY | 0.035 | Distribution quality |
| 4 | GK_DEFENSIVE_TOUCHES_OUTSIDE_OWN_BOX | 0.031 | Sweeper activity |
| 5 | origin_median | 0.026 | Origin league strength |

**Key insight:** Feet matter as much as hands. Passing accuracy and low pass quality rank in the top 5 — modern goalkeepers who can play out from the back are measurably more likely to progress.

**Why GK_PREVENTED_GOALS_* scores mostly = 0 MI:** These metrics are only available from the 2025–2026 season onwards. For the majority of keepers in the dataset (older seasons), the values are missing, making the columns near-constant after imputation. This is a data availability limitation, not a reflection of the metric's true importance.

## Results

### y_plays (PLAYS vs rest)

| Model | AUC-ROC | Avg Precision | F1 |
|-------|---------|---------------|----|
| **Logistic Regression** | **0.623 ± 0.105** | **0.351 ± 0.145** | **0.327** |
| Random Forest | 0.609 ± 0.052 | 0.266 ± 0.030 | 0.000 |
| XGBoost | 0.530 ± 0.093 | 0.216 ± 0.056 | 0.027 |
| Random baseline | 0.500 | 0.170 | — |

### y_up (PLAYS + BENCH vs rest) — main model

| Model | AUC-ROC | Avg Precision | F1 |
|-------|---------|---------------|----|
| Random Forest | 0.669 ± 0.060 | 0.402 ± 0.082 | 0.169 |
| **Logistic Regression** | **0.640 ± 0.098** | **0.488 ± 0.156** | **0.431** |
| XGBoost | 0.630 ± 0.092 | 0.364 ± 0.095 | 0.259 |
| Random baseline | 0.500 | 0.211 | — |

**Why Logistic Regression is the final model:**
XGBoost achieved Train AUC: 1.000 vs Val AUC: 0.530 — a gap of 0.470 indicating severe overfitting on the 174-sample training set. Random Forest never predicted the positive class at the 0.5 threshold (F1 = 0.000 for y_plays). Logistic Regression is the only model that genuinely generalised.

> The y_up model identifies **58% of transfer-ready keepers** (recall) and ranks a genuine talent above a non-talent **64% of the time** (AUC), consistently better than random.

## Scouting App

A local web app that runs predictions on the full dataset.

### Running the app

```bash
# Install dependency
pip install flask

# Start the server
python app.py

# Open in browser
# http://localhost:5000
```

### Features

**Rankings tab**
- All 218 goalkeepers ranked by predicted probability of moving up
- Filter by status (STAYED / PLAYS / BENCH / DROPPED)
- Search by name or player ID
- Click any keeper for a full scouting report showing probability, confidence, top factors for/against, and whether the model was correct

**Compare tab**
- Type two goalkeeper names (autocomplete dropdown after 2 characters)
- Side-by-side prediction cards with probability, verdict, and top factors
- Overlaid radar chart showing both keepers' normalised feature profiles
- Feature-by-feature bar comparison across all 20 features

## Limitations

- **Small dataset:** Only 218 keepers have ≥3 qualifying matches. Per-fold CV sets contain ~7–8 positive examples, making metrics volatile (std > 0.10 on AUC-ROC is expected).
- **Shot-stopping data:** GK_PREVENTED_GOALS_* metrics are only available from the 2025–2026 season. These are conceptually the strongest shot-stopping features but are missing for most keepers.
- **Age feature:** Age is calculated as of March 2026, not at the time of transfer. Consider computing `age_at_transfer` from `birthdate` and `origin_season` for a more meaningful feature.
- **No tracking data:** Positioning, technique, and 1v1 decision-making require tracking data or video analysis — not available in this dataset.
